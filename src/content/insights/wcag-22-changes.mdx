# WCAG 2.2 is here: the changes that actually matter for your website

WCAG 2.2 became a W3C Recommendation in October 2023 and has since been adopted as the
baseline standard by regulators and procurement teams across the UK and EU. If your
accessibility statement still references 2.1, it's time to update more than the version number.

## What's new in 2.2

WCAG 2.2 adds nine new success criteria. Not all of them will affect every product, but
several have broad implications:

### 2.4.11 Focus Appearance (AA)

Focus indicators must now meet a minimum size and contrast ratio. The days of relying on a
thin dotted outline are over. Expect design work to ensure every interactive element has a
visible, high-contrast focus ring.

### 2.5.7 Dragging Movements (AA)

Any functionality that uses a dragging gesture must also be achievable with a single pointer
action (click or tap). Drag-to-sort, sliders, and map interactions are all in scope.

### 2.5.8 Target Size (Minimum) (AA)

Interactive targets must be at least 24×24 CSS pixels, or have sufficient spacing around
them. This particularly affects mobile interfaces and dense data tables.

### 3.2.6 Consistent Help (A)

If a help mechanism (contact link, chat, phone number) appears across multiple pages, it must
appear in the same relative location on each page.

## What to do next

Audit against the new criteria before your next major release. The most common gaps we find
are focus visibility and target sizing on mobile — both of which require design and engineering
collaboration to resolve correctly.
